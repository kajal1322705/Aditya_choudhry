Building a Command Palette with Vanilla JS

November 10, 2025 | 8 min read

JavaScript Web Dev Tutorial

In modern web development, user experience is paramount. One feature that has become increasingly popular, especially in developer-focused tools and websites, is the command palette. It provides a fast, keyboard-driven way for users to access commands and navigate a site. In this article, we'll walk through how to build one from scratch using only vanilla JavaScript, HTML, and CSS.

Why Build a Command Palette?

Command palettes offer several benefits:

  • Speed: Users can access any feature without navigating through menus
  • Discoverability: All actions are searchable in one place
  • Keyboard-first: Perfect for power users who prefer keyboard navigation
  • Accessibility: Improves navigation for users who prefer keyboard interactions

Step 1: The HTML Structure

First, we need a basic structure for our palette. This will typically be an overlay that covers the page, with a container for the input and the results list.

<div class="command-palette-overlay" id="commandPaletteOverlay">
    <div class="command-palette" id="commandPalette">
        <div class="command-palette-header">
            <i class="fas fa-search"></i>
            <input type="text" id="commandInput" 
                   placeholder="Type a command or search...">
        </div>
        <ul class="command-palette-results" id="commandResults"></ul>
    </div>
</div>

Step 2: Styling with CSS

The palette should be visually distinct and easy to use. We'll use a fixed position to keep it on top of other content:

.command-palette-overlay {
    position: fixed;
    top: 0;
    left: 0;
    width: 100%;
    height: 100%;
    background: rgba(0, 0, 0, 0.5);
    backdrop-filter: blur(5px);
    z-index: 2000;
    display: flex;
    justify-content: center;
    align-items: flex-start;
    padding-top: 15vh;
    opacity: 0;
    visibility: hidden;
    transition: opacity 0.3s, visibility 0.3s;
}

.command-palette-overlay.visible {
    opacity: 1;
    visibility: visible;
}

Step 3: The JavaScript Magic

Here's the core JavaScript class that powers our command palette:

class CommandPalette {
    constructor() {
        this.commands = [];
        this.overlay = document.getElementById('commandPaletteOverlay');
        this.input = document.getElementById('commandInput');
        this.results = document.getElementById('commandResults');
        this.selectedIndex = 0;
        
        this.init();
    }
    
    init() {
        this.registerCommands();
        this.bindEvents();
    }
    
    registerCommands() {
        this.commands = [
            { name: 'Go to Home', action: () => window.location.hash = '#home' },
            { name: 'Go to Projects', action: () => window.location.hash = '#projects' },
            { name: 'Toggle Theme', action: () => document.body.classList.toggle('light-mode') },
        ];
    }
    
    bindEvents() {
        document.addEventListener('keydown', (e) => {
            if ((e.ctrlKey || e.metaKey) && e.key === 'k') {
                e.preventDefault();
                this.toggle();
            }
        });
        
        this.input.addEventListener('input', () => this.search());
    }
}

Keyboard Navigation

Proper keyboard navigation is crucial for a good command palette experience:

  • ↑ / ↓ - Navigate through results
  • Enter - Execute selected command
  • Escape - Close the palette
  • Ctrl + K - Open/close the palette

Conclusion

Building a command palette from scratch gives you complete control over its functionality and styling. You can extend it with features like recently used commands, command categories, and nested subcommands.
